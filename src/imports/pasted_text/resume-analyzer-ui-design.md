Build a modern, production-ready frontend for an AI-powered Resume Analyzer and ATS Optimizer web app.

The app should feel like a premium SaaS product used by developers and job seekers.

Design style:
- Dark theme with deep neutral tones (#0f1117, #151922)
- Accent colors: subtle electric blue and soft purple gradients
- Minimal, clean, and calm — avoid flashy or overly futuristic styles
- Use clean typography (Inter or similar)
- Prioritize readability and spacing over decoration

UX principles:
- Focus on clarity and hierarchy
- Every section should have a clear purpose
- Avoid clutter and unnecessary elements
- Use subtle animations and transitions

Interaction style:
- Include tooltip and popover placeholders for Floating UI integration
- Hover states should feel responsive and natural
- Avoid excessive animation

Important:
- The UI should not look like a generic template or AI-generated layout
- Introduce slight asymmetry and natural spacing variations
- Make it feel handcrafted and intentional

Pages required:
- Landing page
- Dashboard
- Resume editor
- Insights page

Output should be clean, modular, and ready for React integration

Design a modern SaaS landing page for an AI Resume Analyzer.

Hero section:
- Headline: "Your Resume, Decoded."
- Subheadline explaining ATS scoring and AI suggestions
- Drag-and-drop resume upload area as the main focus
- Role selection dropdown (Software Engineer, Data Analyst, etc.)

Design details:
- Centered layout with strong visual hierarchy
- Upload area should look interactive (glass card, dashed border)
- Include a primary CTA button: "Analyze Resume"

Interaction placeholders:
- Add tooltip near upload area explaining supported formats (PDF, DOCX)
- Add hover states for buttons and upload zone

Additional sections:
- Features (ATS scoring, AI suggestions, keyword analysis)
- Simple 3-step workflow (Upload → Analyze → Improve)
- Footer with minimal links

Style:
- Clean, modern, slightly asymmetrical
- Avoid overly common SaaS layouts

Design a dashboard for a resume analysis application.

Layout:
- Left sidebar with icons and labels (Dashboard, Editor, Insights, Settings)
- Main content area with modular cards

Top section:
- Large ATS score display (circular or clean numeric emphasis)
- Short explanation text below score

Main cards:
- Skills match percentage
- Missing keywords (tag-style chips)
- Resume weaknesses
- Improvement suggestions

Interaction placeholders:
- Each card should include an info icon
- On hover, a tooltip should appear (Floating UI integration)

Design details:
- Cards should have subtle shadows and depth
- Hover effects: slight elevation and glow
- Maintain spacing and avoid clutter

Feel:
- Analytical, calm, professional

Design a split-screen resume editor interface.

Layout:
- Left panel: original resume content
- Right panel: AI-enhanced version

Features:
- Highlight improved text in the AI version
- Inline suggestions for rewriting bullet points
- "Accept change" and "Reject" actions

Interaction placeholders:
- Hover over highlighted text → show floating suggestion popover
- Contextual rewrite suggestions near selected text (Floating UI)

Design style:
- Clean and distraction-free
- Focus on readability
- Use subtle highlights (not bright or neon)

Feel:
- Like a professional writing assistant, not an AI gimmick

Design an insights page that explains resume performance.

Sections:
- Why your resume gets rejected
- Missing keywords for selected role
- Suggestions to improve

UI:
- Expandable sections or accordion layout
- Clean typography-focused design

Interaction placeholders:
- Hover over technical terms (ATS, keyword density) → show tooltip
- Smooth transitions between expanded sections

Style:
- Editorial, clean, slightly different from dashboard to add variety

Create a reusable component system for this application.

Components:
- Buttons (primary, secondary, ghost)
- Cards with hover states
- Input fields
- Dropdown menus
- Tooltips and popovers (important for Floating UI)
- Modals

Interaction behavior:
- Tooltips should feel lightweight and contextual
- Dropdowns should appear near trigger elements (Floating UI ready)
- Smooth transitions, no abrupt animations

Design rules:
- Consistent spacing (8px system)
- Rounded corners (8–12px)
- Subtle borders and shadows

Important:
- Components should feel cohesive but not overly uniform

Refine the UI to feel more human-designed and less template-based.

Adjust:

- Vary spacing to avoid perfect grid rigidity
- Reduce repetitive patterns across sections

Improve:
- Visual hierarchy using spacing instead of color
- Subtle differences between sections

Ensure:
- The UI feels like it was designed by a human product designer
- Avoid overly polished or generic SaaS patterns

Keep:
- Cleanliness
